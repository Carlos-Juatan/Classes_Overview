    Antes de fazer um upgrade ou downgrade desligar a máquina antes

Configurando o Firewall
  • Aula 01 - Visão Geral do Firewall

    A importancia de ter um firewall ativo

    O firewall tradicional opera diretamente na sua VPS e, ao ser confrontado com ataques como bots que enviam uma quantidade massiva de requisições, consome os recursos da sua máquina, como CPU e memória. Isso é um problema, especialmente se você está tentando economizar e já está no limite do desempenho da sua VPS.

  • Aula 02 - O que é edge firewall

    o edge firewall é uma solução na nuvem que se concentra em proteger a sua infraestrutura antes que as requisições cheguem à sua VPS. Ele é configurado diretamente no provedor de serviços, como DigitalOcean ou Hetzner, permitindo que esse fornecedor gerencie as tentativas de invasão e gaste os recursos da nuvem em vez dos seus. Isso não só melhora a segurança, mas também otimiza a performance da sua VPS.

  • Aula 03 - Configurando o Firewall no Digital Ocean

    Não pretendo usar o Digital Ocean

  • Aula 04 - Configurando o Firewall no Hetzner

    Comece a configurar o seu firewall apagando todas as regras que vem nele

    Crie uma regra do TCP porta 22 e cole os IPV4s das máquinas do seu servidor e o do seu próprio computador

    Crie mais 2 regras uma para o HTTP e HTTPS usando as portas 80 e 443 e deixe aberto para todos colocando os IPV4 e IPV6

    Portas obrigatórias para configurar o Firewall


    Porta TCP 22: Porta que permite acessar SSH
    Porta TCP 80: Porta Http
    Porta TCP 443: Porta Https / SSL
    Porta TCP 2377: Para comunicação entre workers e managers
    Porta TCP 2376: TCP para comunicação entre workers e managers
    Porta TCP 7946: Para descoberta de nodes de rede de Overlay
    Porta UDP 7946: Para descoberta de nodes de rede de Overlay
    Porta UDP 4789: Rede Overlay do Docker Swarm
    Porta TCP 9001: Porta interna do Portainer Agent
    Porta TCP 9443: Porta interna do Portainer Agent com SSL


    Portas Opcionais para configurar no Firewall


    Porta TCP 3306: Porta para MySQL para Acesso Remoto
    Porta TCP 5432: Porta para Postgres para Acesso Remoto
    Porta TCP 5672: Porta para o RabbitMQ para Acesso Remoto
    Porta TCP 6379: Porta para o Redis para Acesso Remoto
    Porta TCP 27017: Porta para o MongoDB

    Terminando de configurar, desça a página até a opção Apply To marque e selecione o servidor

Acesso via SSH da VPS com Termius
  • Aula 01 - Visão Geral do Termius

    Tem uma versão gratuita, mas para trabalhar disso o ideal é ter o plano pago

    Caso queira o plano pago, pegue o plano movel que é muito mais barato

  • Aula 02 - Configurações Gerais do Termius

    Você pode fazer login em uma conta gratuita, para deixar salvo suas configurações e preferências

    Nas configurações de Terminal habilitar o autocomplete

    Na aba lateral Theme colocar em auto ou dark mode

  • Aula 03 - Cadastrando Senhas do Termius

    Na aba lateral Keychain

    No menu de cima ao clicar em KEY selecione New Identity

    Então Nomeie a credencial para algo que esteja alinhado com o projeto, coloque o root clique em mais e adicione um password e nele digite a senha do seu servidor

  • Aula 04 - Configurando Acesso SSH de uma VPS

    Na aba lateral Hosts

    No menu de cima ao clicar em HOST selecione New Host

    Coloque o nome ou o IPv4 do servidor, dê um nome para esse acesso ( de preferência o mesmo nome do servidor ), coloque tags para ficar mais fácil de achar esse acesso depois, e em SSH on coloque a porta do acesso SSH que é 22 por padrão, e clique em See More e selecione a credencial que você criou anteriormente

    Ao clicar para acessar ele vai pedir o Fingerprint que nem pede no terminal é só adicionar e continuar

  • Aula 05 - Configurando Acesso SFTP de uma VPS

    Na aba lateral SFTP é um recurso de gerenciamento de arquivos entre server e host, permitindo a transferência de aquivos só clicando e arrastando para as pastas

  • Aula 09 - Acesso SSH via Windows 11

    Na loja do Windows instalar o Terminal

    No menu inicial pesquisar por Recursos Opcionais então pesquisar por SSH e instale ambos o Cliente OpenSSH e o Server OpenSSH

    Ao finalizar a instalação do Cliente OpenSSH será possível usar o comando SSH vai estar disponivel no Terminal do windows ( O comando é SSH [nome do root]@[IPv4 do servidor] )

    Ao digitar o comando ex:root@5.161.234.171 confirme o Fingerprint digitando yes no terminal e por fim a senha do servidor, assim você terá acesso ao servidor usando o windows

  • Aula 03 - Configurando o Cloudflare

    Adicione o domínio que você quer usar

    Insira seu domínio

    Escolha o plano gratuito

    Ele vai fazer uma varredura no seu domínio e adicionar os registros DNS que ele encontrar

    Depois disso, ele vai informar o seu name server, você provavelmente estará usando um domínio BR, então vai no Registro Br informar que você usará esse Domínio no Cloud Flare, se o seu Domínio for novo, ok. Mas se ele tiver migrando o domínio para o cloud flare, o Registro Br demora 3 hrs para fazer qualquer mudancça de Domínio. ( Isso é uma limitação do próprio domínio br )

    Depois do Registro Br liberar para trocar os Nameservers ( Só libera depois de 3 hrs Caso tenha feito a migração ), então troque os Nameservers do Registro Br usando os do Cloud Flare
