Configurando o Firewall
  • Aula 01 - Visão Geral do Firewall

    A importancia de ter um firewall ativo

    O firewall tradicional opera diretamente na sua VPS e, ao ser confrontado com ataques como bots que enviam uma quantidade massiva de requisições, consome os recursos da sua máquina, como CPU e memória. Isso é um problema, especialmente se você está tentando economizar e já está no limite do desempenho da sua VPS.

  • Aula 02 - O que é edge firewall

    o edge firewall é uma solução na nuvem que se concentra em proteger a sua infraestrutura antes que as requisições cheguem à sua VPS. Ele é configurado diretamente no provedor de serviços, como DigitalOcean ou Hetzner, permitindo que esse fornecedor gerencie as tentativas de invasão e gaste os recursos da nuvem em vez dos seus. Isso não só melhora a segurança, mas também otimiza a performance da sua VPS.

  • Aula 03 - Configurando o Firewall no Digital Ocean

    Não pretendo usar o Digital Ocean

  • Aula 04 - Configurando o Firewall no Hetzner

    Comece a configurar o seu firewall apagando todas as regras que vem nele

    Crie uma regra do TCP porta 22 e cole os IPV4s das máquinas do seu servidor e o do seu próprio computador

    Crie mais 2 regras uma para o HTTP e HTTPS usando as portas 80 e 443 e deixe aberto para todos colocando os IPV4 e IPV6

    Portas obrigatórias para configurar o Firewall


    Porta TCP 22: Porta que permite acessar SSH
    Porta TCP 80: Porta Http
    Porta TCP 443: Porta Https / SSL
    Porta TCP 2377: Para comunicação entre workers e managers
    Porta TCP 2376: TCP para comunicação entre workers e managers
    Porta TCP 7946: Para descoberta de nodes de rede de Overlay
    Porta UDP 7946: Para descoberta de nodes de rede de Overlay
    Porta UDP 4789: Rede Overlay do Docker Swarm
    Porta TCP 9001: Porta interna do Portainer Agent
    Porta TCP 9443: Porta interna do Portainer Agent com SSL


    Portas Opcionais para configurar no Firewall


    Porta TCP 3306: Porta para MySQL para Acesso Remoto
    Porta TCP 5432: Porta para Postgres para Acesso Remoto
    Porta TCP 5672: Porta para o RabbitMQ para Acesso Remoto
    Porta TCP 6379: Porta para o Redis para Acesso Remoto
    Porta TCP 27017: Porta para o MongoDB

    Terminando de configurar, desça a página até a opção Apply To marque e selecione o servidor

  • Aula 04 - Instalação e Configuração do Docker

    Abra o SSH do seu manager usando o Termius e cole o comando.


    bash - Docker GPG key

    
apt-get update
apt install -y sudo gnupg2 wget ca-certificates apt-transport-https curl gnupg nano htop
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

    Depois copie e cole o código, para definir os recursos do Docker.


    bash - Docker Apt sources

    
echo \
  "deb [arch="$(dpkg --print-architecture)" signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  "$(. /etc/os-release && echo "$VERSION_CODENAME")" stable" |
  sudo tee /etc/apt/sources.list.d/docker.list >/dev/null
sudo apt-get update

    Depois copie e cole o código para instalar o Docker.


    bash - Docker Install

    
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

    Depois copie e cole o código, para coloca o docker para iniciar com o sistema.


    bash - Docker como serviço

    
sudo systemctl start docker.service

    Depois copie e cole o código, para coloca o container id para iniciar com o sistema.


    bash - container id iniciar com o sistema

    
sudo systemctl enable containerd.service
  • Aula 05 - Ativação do Modo Swarm no Docker

    Nota: onde tiver 000000 trocar para o ip do SSH do manager


    bash - Token do Swarm

    Para adicionar mais um manager copie e cole o código que aparecer no terminal



    Agora para adicionar uma rede para o Swarm

                                                docker network create --driver=overlay network_swarm_public
  • Aula 07 - Configurando o Traefik 2

    Abra o Termius na barra lateral selecione o SFTP abra a pasta onde está o stack do traefik 2 então escolha o servidor Manager, você só fará essa configuração no Manager

    Depois de escolher o servidor Manager clique e arrate o arquivo traefik-v2.yaml para dentro do SSH

    Depois de colar o arquivo na aba lateral vá em Hosts e entre no Teminal do SSH Manager e cole o comando para fazer o download da imagem do traefik v2


    bash - Download Traefik 2

    
docker pull traefik:v2.11.3

    Agora copie e cole o comando para executa o stack do Traefik v2


    bash - Executa o stack do Traefik v2

    
docker stack deploy --prune --detach=false --resolve-image always -c traefik-v2.yaml traefik

    docker stack deploy = comando para executar o stack do docker

    --prune = para remover os serviços que não estão mais em uso. É o mesmo comando tanto para lançar como para atualizar
    ( Se não exitir ele cria e se existir ele autaliza )

    --detach=false = para não rodar em segundo plano
    ( Para poder ver em tempo real todas as informações na tela )

    --resolve-image always = para sempre baixar a imagem mais atualizada

    -c = para indicar o arquivo de configuração
    ( Que no caso é o traefik-v2.yaml )

    traefik-v2.yaml = nome do arquivo de configuração

    traefik = nome do stack
    ( Nome que ele vai ficar no docker )

  • Aula 08 - Atualizando o Traefik 2

Setup N8N
  • Aula 04 - Configurando o DNS do N8N

    Não use o nome n8n ou qualquer nome de serviço como link de endereço para sua aplicação, é questão de segurança não usar os nomes de serviço pois é mais fácil dos boots acharem


    Para o editor use workflows

    E para o n8n webhooks use webhooks

    Para saber o endereço já propagou use o site mxtoolbox.com

    No canto superior direito tem uma opção all Tools clique nela

    Em seguida escolha a opção cname

    Após isso, digite o endereço que você acabou de criar no Cloudflare

    Se ele achar o endereço, quer dizer que ele já propagou e você já pode fazer o restante do processo

    Se estiver escrito "DNS Record Not Published" e "DNS Record Not Found" não faça o deploy do stack ele vai tentar gerar o SSL e não vai funcionar, e isso vai causar punição de tempo onde na 5° tentativa fracassada vai ter que esperar 1 dia para tentar de novo

  • Aula 05 - Variáveis de Ambiente do N8N

    Para não dá problema quando precisar atualizar, os stacks estão separados, e quando for nescessário atualizar todos os stacks para não dá erro, atualizar primeiro o Editor, depois o Webhook e depois o Worker


    Editando o n8n Editor a variável que você não pode perder é a N8N_ENCRYPTION_KEY pois, é responsável pela criptografia com o banco de dados, se perder ela não será mais possível acessar o n8n

    Para gerar uma nova senha pode usar o avast random password generator usando pelo menos 20 caracteres


    Essa senha tem que ser a mesma em todos os stacks do n8n


    A variável N8N_HOST usa o endereço do workflows sem o http e sem a barra no final, já as variáveis N8N_EDITO_BASE_URL e WEBHOOK_URL, precisam do http e da barra no final

    Nos labels do traefik mude o endereço do editor para o que será usado na instalação

  • Aula 06 - Configurando Usuários no N8N

    Para o uso de multiplos usuários no n8n se usa o serviço smtp2http que transforma a requisição smtp em um webhook para poder ser usado no n8n "burlando" a norma de envio de email que é rigorosa

  • Aula 07 - Configurando os Webhooks do N8N

    O n8n webhooks pode e deve ter mais de uma replica rodando, dependendo dos casos uma maquina só para o manager outra só para o webhook e outra só para o workers, isso vai depender das escalabilidade do projeto


    Para cada replica as configurações de espaço usado é para cada uma das réplicas, e não o total delas. ou seja se tiver 1 cpu, 1024M e 2 réplicas então será 2 cpus e 2048M pois é para cada réplica

    A única configuração no label que precisará ser feita é no traefik.http.routers.n8n.webhooks.rulecolocando o endereço do webhooks

    Todas as configuraçãos de environment devem ser copiadas do n8n-editor

  • Aula 08 - Configurando o Worker do N8N

    O worker é o responsável por executar os fluxos de trabalho, ele não tem acesso ao banco de dados, e não tem acesso ao editor, ele só executa os fluxos de trabalho


    A concurrency é um valor para rodar processos em paralelo e ele também é por replica, se voce colocou 10 de concorrencia e tem 2 réplicas, então o total são 20 processos paralelos

    No caso do worker fazer replicas é só em ultimo caso, primeiro almenta a memória e cpu se não resolver almentar a concorrência e se mesmo assim não resolver ai sim almentar as réplicas

  • Aula 09 - Criando o Banco de Dados do N8N

    Antes de instalar o n8n precisa fazer a instalação do banco de dados


    No caso, tendo o Postgres já instalado, vá no portainer, acesse a aba container e acesse o serviço do Postgres depois, clique em console, para abrir o terminal digite em user postgres ( tudo minúsculo ) e depois em conectar, dessa forma não será nescessário informar o usuário e senha do postgres

    Depois rode o comando bash


    bash - Acessando o Postgres

    
psql

    No n8n foi configurado um banco de dados e ele precisa ser criado antes da instalação

    No caso o nome do banco de dados é n8n_queue, então use o comando CREATE DATABASE n8n_queue;


    bash - Criando o banco de dados

    
CREATE DATABASE n8n_queue;

    Cada banco de dados deve ter um nome diferente e não pode ter caracteres especiais

  • Aula 10 - Deploy do N8N no Swarm

    Comece pelo n8n editor copie o stack e cole no portainer, renomeie o container para o mesmo nome do serviço e então dê o deploy


    O editor precisa rodar as migrations antes de continuar o processo de instalação


    Para verificar se as migrations foram realizadas com sucesso, vá na aba services e espere o n8n editor termianr de baixar as imagens e está rodando, quando isso acontecer, vá na aba containers e selecione o n8n editor e abra os logs, e verifique se o processo de migrations foi finalizado. Somente quando ele estiver finalizado e pronto pra usar que deve continuar a instalação do n8n

    Isso também vale para atualização


    Agora siga copiando o stack do webhook e perceba que como a imagem do n8n já havia sido baixada a instalação do webhook será muito mais rápida


    Siga então para a isntalação do worker copiando e colando o stack no portainer

  • Aula 03 - Variáveis de Ambiente Gerais

  • Aula 04 - Autenticação da Evolution API

    A Evolution API 2 possui uma chave pública, crie uma chave forte usando o Avast password generator com pelo menos 20 ou 30 caracteres, o professor recomenda não usar caracteres especiais, ou você pode usar um uuid para gerar a senha

    ( No geral se for para enviar o link para o cliente use o avast e se não use o uuid )


    A variável - AUTHENTICATION_EXPOSE_IN_FETCH_INSTANCES=true se for para somente 1 cliente pode habilitar mostrar a senha, caso contrário, se for uma Evolution SaaS para muitos usuários acessarem a api não, habilitar de geito nenhum, pois cada cliente poderar ver a senha dos outros clientes

  • Aula 05 - Configurações do Banco de Dados

    Vamos ter um postgres e um redis para o n8n e outro postgres e redis para a restante das ferramentas


    Primeiro pegue o login e senha so seu banco de dados e copie no no código da stack da Evolution

    O usuário do postgres é o nome do serviço

    A SENHA você substitue pela sua senha e o usuário fica depois da senha@

    A ultima informação nessa linha de código é qual o banco de dados dentro do postegres

  • Aula 06 - Configurando Filas e Webhooks

    O webhook é em tempo real, ou seja se perder perdeu, mas o modo fila não é em tempo real e pode ser acessado a qualquer momento por isso é o modo mais indicado de usar


    É recomendado usar o rabbitMQ para formar as filas ao inves de usar o webhook, principalmente se for usar o n8n

    A parte - RABBITMQ_URI=amqp://usuario:senha@rabbitmq:5672/default é seguida por usuário, senha, o nome do serviço no docker, a porta, e o virtual host ( default )


    Tanto para o nome do banco de dados como para o nome da exchange do rabbitMQ o ideal é usar o mesmo nome do seviço da evolution só para deixar padronizado, principalmente se usar mais de uma evolution

  • Aula 09 - Configurações de Cache da Evolution API

    Para configurar o redis na linha - CACHE_REDIS_URI=redis://redis:6379/1, a estrutura é redis://nome do serviço no docker : a porta / banco de dados

    Se você tiver mais de uma instalação do evolution cada uma vai ficar em um banco de dados diferente, uma no 1, outra no 2 e assim por diante

  • Aula 12 - Sobre o Deploy da Evolution

    O endereço do label no código - traefik.http.routers.evolution_v2.rule=Host(`evo.agenciasynca.com.br`) o endereço deve ser trocado pelo mesmo endereço do código - SERVER_URL=https://evo.agenciasynca.com.br

    E lembrando que nesses labels a parte evolution_v2 é o nome do serviço, e se esse nome do serviço for trocado também precisa trocar nos labels

  • Passo 01 - Instalando o Docker

  • Passo 02 - Escolha um Domínio Falso Local:

  • Passo 03 - Configurar a Resolução de Nomes Local (/etc/hosts)

    Para que seu computador saiba que codesefaz.local e seus subdomínios (como painel.codesefaz.local ( para o portainer ), ou workflows.codesefaz.local ( para o editor do n8n ) ) apontam para o seu próprio computador (localhost), você precisa editar o arquivo de hosts do seu sistema operacional.


    No Windows: Edite o arquivo C:\Windows\System32\drivers\etc\hosts ( precisa de permissões de administrador ).

    Adicione as seguintes linhas (ou similar, adaptando para seus subdomínios):


    Arquivo hosts aberto no bloco de notas


    127.0.0.1 codesefaz.local ( para o servidor central )
    127.0.0.1 painel.codesefaz.local ( para o portainer )
    127.0.0.1 workflows.codesefaz.local ( para o n8n editor )
    127.0.0.1 webhooks.codesefaz.local ( para o n8n webhook )
    127.0.0.1 baserow.codesefaz.local ( para o baserow talvez melhor trocar o nome )
    127.0.0.1 evo.codesefaz.local ( para a evolution api )

    No windows para conseguir salvar o arquivo depois de modificar você precisar abrir primeiro o bloco de notas em modo administrador e depois atravez do bloco de notas abrir a pasta e o arquivo, só assim será possível fazer as mudanças pois é um sistema de segurança do windows
